Project Case Study

I've put a little case study together to give you an idea of how I work on each project. The following case is a project for a petrol station which had a deadline of 2 weeks.

Project Plan

The project started off with the establishment of a charter and project analysis. Project goals and deadlines were established along with a defined sitemap. Technical and creative requirements were decided, e.g. responsive breakpoints for device compatibility, and organisational perspectives were decided upon. Site content and a strategy for search engine optimisation (SEO) were also established.

Design

Wireframes were prepared to convey page layout. UX storyboards were then created to show the flow of the website. These storyboards contain all aspects of the sites visual design in order to show how the completed site will look and to assist in the process of site development.

Development

Development of the site used HTML5, CSS3 and jQuery/Javascript technologies, respectively, to build the site content, style and behaviour. It's at this stage when all responsive design was also implemented to enable for a better browser experience on mobile devices.

Validation

Completed code was validated according to W3C standards and tested for accessibility. The site was tested across multiple browsers and devices before launch. You can check out the completed site here.
